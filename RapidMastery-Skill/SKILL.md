# Rapid Mastery - 主题速通专家

快速、科学、深度地掌握任何主题。基于费曼学习法、第一性原理、间隔重复和刻意练习等认知科学理论设计。

## 能力

本 Skill 帮助你：

1. **解构复杂主题** - 用第一性原理拆解为可管理的核心概念
2. **检验真实理解** - 通过费曼对话暴露理解盲区
3. **高效记忆巩固** - 生成间隔复习计划，对抗遗忘曲线
4. **针对性强化** - 诊断薄弱点，生成刻意练习
5. **实战应用** - 设计渐进式项目，从知道到做到

## 使用方式

### 启动 Skill

```
/rapid-mastery <主题> [目标深度] [时间预算]
```

**参数说明：**
- `<主题>`: 必填。想学习的任何主题（如"Python装饰器"、"区块链"、"光合作用"）
- `[目标深度]`: 可选。`know`(了解) | `apply`(应用) | `teach`(教授) | `master`(精通)。默认 `apply`
- `[时间预算]`: 可选。`1h` | `1day` | `1week`。默认 `1day`

### 交互模式

启动后，Skill 将引导你完成 7 步学习循环：

```

## 工作流程详解

### Step 1: 目标锚定 (Goal Anchoring)

**目的**: 明确学习边界，避免范围蔓延

**系统行为**:
- 询问当前水平（完全新手/有基础/想深化）
- 确认应用场景（考试/工作项目/兴趣爱好/教别人）
- 根据时间预算裁剪内容深度

**输出示例**:
```

主题: Python 装饰器
当前水平: 有 Python 基础，写过函数
目标深度: 能在项目中实际应用
时间预算: 3 小时
应用场景: 编写可复用的日志/缓存/权限控制
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 2: 原理解构 (First Principles Deconstruction)

**目的**: 将复杂主题拆解为 5-7 个核心概念，建立认知地图

**系统行为**:
- 识别主题的第一性原理
- 生成层级化概念图谱
- 为每个概念提供生活化类比

**输出示例**:
```
🧬 概念解构: Python 装饰器
━━━━━━━━━━━━━━━━━━━━━━━━━━━
核心概念树:

装饰器
├── 1. 函数是一等公民 (Functions as First-Class Citizens)
│   └── 类比: 函数就像数据一样可以被传递
├── 2. 高阶函数 (Higher-Order Functions)
│   └── 类比: 就像"函数加工厂"，进一个函数，出一个新函数
├── 3. 闭包 (Closure)
│   └── 类比: 函数记得自己被创建时的环境，像带记忆的背包
├── 4. @语法糖 (@ Syntax)
│   └── 类比: 装修时的"快捷指令"，省得写冗余代码
└── 5. 装饰器模式应用场景
    ├── 日志记录 (Logging)
    ├── 性能计时 (Timing)
    ├── 缓存 (Caching)
    └── 权限检查 (Authorization)

学习顺序建议: 1 → 2 → 3 → 4 → 5
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 3: 费曼讲解 (Feynman Teaching)

**目的**: 用简单语言解释，暴露理解盲区

**系统行为**:
- 扮演"聪明但一无所知的小学生"
- 要求用户用一句话解释概念
- 追问"为什么"、"如果不这样会怎样"、"和XX有什么区别"
- 直到用户能清晰解释每个核心概念

**交互示例**:
```
🎓 费曼挑战: 解释"闭包"
━━━━━━━━━━━━━━━━━━━━━━━━━━━
假设我是高中生，没学过编程。请用一句话告诉我：
"闭包是什么？为什么需要它？"

[用户回答后]

追问 🤔:
1. 你说函数"记住"变量，是像人记住事情一样吗？
2. 如果我在函数外面修改那个变量，函数里"记住"的会变吗？
3. 这和全局变量有什么区别？

请用具体代码示例说明你的理解。
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 4: 双编码学习 (Dual Coding)

**目的**: 通过多种形式加深记忆（文字+视觉+代码+类比）

**系统行为**:
- 生成 ASCII 流程图或文字图示
- 创建 3 个以上生活化类比
- 提供最小可运行代码示例
- 生成记忆口诀或故事

**输出示例**:
```
🎨 多维度理解: 装饰器执行流程
━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 流程图:

   定义装饰器函数
         ↓
   定义被装饰函数 ←──── 这里还没执行！
         ↓
   @decorator 语法糖
         ↓
   实际发生: my_func = decorator(my_func)
         ↓
   调用 my_func() ──────→ 先执行装饰器内部逻辑
         ↓                   ↓
   返回 wrapper 结果 ←── 再执行原函数逻辑

🧩 类比集:
1. 手机壳: 装饰器就像手机壳，不改变手机本身，但增加了保护功能
2. 快递员: 原函数是包裹，装饰器是快递员，可以在送货前后做额外事情
3. 餐厅服务: 原函数是厨师做菜，装饰者是服务员，负责上菜、收盘、记账

💻 最小示例:
```python
def my_decorator(func):
    def wrapper():
        print("之前做点事")
        func()
        print("之后做点事")
    return wrapper

@my_decorator
def say_hello():
    print("Hello!")

say_hello()
# 输出: 之前做点事 → Hello! → 之后做点事
```

🧠 记忆口诀:
"装饰器，包函数，高阶闭包记心中。
@符号，语法糖，调用逻辑要分清。"
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### Step 5: 主动回忆测试 (Active Recall Quiz)

**目的**: 通过闭卷测试强化记忆，诊断薄弱点

**系统行为**:
- 生成 5-8 道开放式问题（非选择题）
- 要求用户写出完整答案
- 分析答案，指出理解误区
- 给出针对性改进建议

**输出示例**:
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━
请写出你的答案：

Q1: 用你自己的话，解释为什么 Python 中函数可以被当作参数传递？
[等待用户回答]

Q2: 下面代码的输出是什么？为什么？
```python

    print("装饰器执行")
    return func

@decorator
def hello():
    print("Hello")

hello()
```
[等待用户回答]

Q3: 装饰器和普通函数调用有什么区别？什么时候装饰器内的代码会执行？
[等待用户回答]

...

📊 诊断结果:
✅ 概念理解: 优秀
⚠️ 执行时机: 需要加强 - 你似乎混淆了装饰时和调用时
❌ 闭包应用: 薄弱 - 建议重新学习 Step 3 的追问环节
━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**目的**: 整合所有知识，完成真实应用场景

**系统行为**:
- 设计一个完整的小项目
- 要求使用本主题的核心知识点
- 提供项目需求和验收标准
- 可选：提供参考答案

**输出示例**:
```
🚀 实战项目: 构建 Web API 工具集
━━━━━━━━━━━━━━━━━━━━━━━━━━━

【项目需求】
为一个假想的 Web 框架实现以下装饰器：

1. @route('/path') - 路由注册（使用带参数的装饰器）
2. @auth_required - 检查用户是否登录
3. @cache(timeout=60) - 缓存函数结果
4. @timer - 记录函数执行时间并打印

【验收标准】
- 所有装饰器可以叠加使用: @route('/api') @auth_required @cache(60)
- 装饰后的函数保留原函数名和文档
- 提供 3 个 API 端点演示用法

【脚手架】
```python
from functools import wraps

# 你的实现...

# 测试用例
@app.route('/public')
def public_api():
    return "所有人可见"

@app.route('/admin')
@app.auth_required
def admin_api():
    return "仅管理员可见"
```

完成后，你将完全掌握 Python 装饰器！

## 故障排除

**Q: 我觉得 Step 3 的追问太烦人了**
A: 在配置中设置 `"strict_mode": false`，将减少追问次数。

**Q: 生成的项目太简单/太难**
A: 直接告诉系统调整难度："请给我一个更复杂的项目，需要用到异步装饰器"。

**Q: 我想跳过某些步骤**
A: 直接说："跳过费曼环节，直接给我测试题"，或启动时指定：`/rapid-mastery 主题 --skip=feynman`

**Q: 如何复习之前学过的主题？**
A: 使用复习模式：`/rapid-mastery --review 区块链`，将快速过一遍测试并生成新的复习计划。
