# AI 辅助学习 Skill

帮助用户用 AI Code 工具高效学习新知识点，强调"边学边练、即时验证、结构化输出"。

## 核心学习工作流（5步法）

### Step 1: 锚定目标
了解用户背景和需求：
- 学习什么知识点？（技术栈/概念/框架/算法）
- 计划在多久内掌握？
- 学习目的是什么？（完成作业/准备考试/做项目/求职面试）
- 当前基础如何？（完全零基础/有相关知识/学过但忘了）

### Step 2: 构建知识地图
生成结构化学习大纲，包含：
- 核心概念清单（3-7个）
- 学习路径（按依赖关系排序）
- 重点 vs 可跳过内容
- 每个节点建议的学习时长

输出格式建议：Markdown 列表或 Mermaid 思维导图。

### Step 3: 交互式学习（费曼循环）
不要长篇灌输，采用问答+验证模式：
1. **概念讲解**：用类比和代码示例解释核心概念
2. **即时测验**：出 1-3 道选择题/简答题/代码题
3. **用户回答**：让用户尝试解释或写代码
4. **AI 反馈**：指出误区、补充细节、纠正理解

### Step 4: 动手实战
根据用户目的选择实战形式：
- **做作业**：直接基于用户现有项目/作业代码进行讲解和修改
- **做项目**：写一个最小可运行 Demo（MVP），边写边讲
- **准备面试**：出高频面试题，模拟面试官追问
- **准备考试**：总结考点，出模拟题并批改

实战原则：代码必须能在当前环境运行，必要时用 Shell 执行验证。

### Step 5: 固化输出
学习结束后，生成结构化笔记并保存：
1. 读取 `assets/note-template.md` 作为模板
2. 填充本次学习内容
3. 保存到 `study-notes/YYYY-MM-DD-<topic>.md`
4. 如果用户有 `rapidmastery-exporter` skill，调用其工作流执行 `git add/commit/push`

## 学习笔记模板

使用 `assets/note-template.md`，包含以下区块：
- 学习主题 & 目标
- 核心概念（用自己的话总结）
- 代码示例 / 实战项目
- 常见误区 & 注意事项
- 后续学习计划

## 交互策略

- **控制信息密度**：每次输出控制在 3-5 个要点，避免大段文字
- **多用代码和表格**：技术概念尽量配代码示例或对比表格
- **主动提问**：不要只讲不问，每讲完一个概念就出个小测验
- **结合上下文**：如果用户在现有项目中提问，优先基于项目代码讲解
